# Decision: 优先建立基础 Harness

## 背景

项目当前已有项目管理规范，但还没有实现代码、Agent 指令、验证脚本或配套文档。

## 决策

在实现产品功能前，先建立最小可用的项目 harness。

该 harness 包括：

- `AGENTS.md`
- `docs/` 下的基础文档
- `scripts/` 下的统一脚本入口
- 用于测试、示例、fixtures 和 CI 的标准项目目录

## 影响

后续任务可以通过稳定的项目约定进行范围控制、实现、验证和审查。

在实现技术栈确定前，初始脚本是轻量占位脚本。

## 备选方案

- 立即开始功能实现：不采用，因为当前缺少项目结构和验证支撑。
- 在创建任何文件前完整设计架构：不采用，因为小型 harness 可以指导探索，同时避免过早锁定架构。
